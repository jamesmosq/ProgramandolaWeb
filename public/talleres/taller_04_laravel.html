Configurar el archivo .env
DB_* · APP_NAME · APP_LOCALE · .env vs config/
▼

El archivo .env guarda la configuración sensible del entorno (BD, mail, app key). Nunca se sube a Git. Laravel lo lee automáticamente.

🔒
.env está en .gitignore por defecto. Usa .env.example como plantilla para otros desarrolladores. Nunca pongas contraseñas reales en .env.example.
ENV
# Configuración de la plataforma EduCode
APP_NAME="EduCode CEFIT"
APP_ENV=local
APP_KEY=base64:...  # Generado con: php artisan key:generate
APP_DEBUG=true
APP_URL=http://programando.test

APP_LOCALE=es
APP_FALLBACK_LOCALE=es
APP_FAKER_LOCALE=es_Col

# Base de datos MySQL local (WAMP)
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=programando
DB_USERNAME=root
DB_PASSWORD=base1234

# Sesiones y caché en BD (no requiere Redis)
SESSION_DRIVER=database
CACHE_STORE=database
QUEUE_CONNECTION=database

# Mail en desarrollo: solo guarda logs, no envía
MAIL_MAILER=log

Migraciones — versionando la BD
make:migration · up() · down() · migrate · foreignId · constrained
▼

Las migraciones en Laravel son el equivalente a scripts SQL versionados. En vez de ejecutar SQL manualmente, escribes PHP y Laravel se encarga del SQL.

BASH
# Crear migraciones (orden padre → hijo)
php artisan make:migration create_modulos_table
php artisan make:migration create_lecciones_table
php artisan make:migration create_pasos_table
php artisan make:migration create_ejercicios_table
php artisan make:migration create_progresos_table

Migration de módulos

PHP
// database/migrations/..._create_modulos_table.php
public function up(): void
{
    Schema::create('modulos', function (Blueprint $table) {
        $table->id();
        $table->string('nombre', 120);
        $table->text('descripcion')->nullable();
        $table->string('icono', 60)->nullable();
        $table->unsignedTinyInteger('orden')->default(0);
        $table->boolean('activo')->default(true);
        $table->timestamps();
    });
}

public function down(): void
{
    Schema::dropIfExists('modulos');
}

// Migration de lecciones (con FK)
public function up(): void
{
    Schema::create('lecciones', function (Blueprint $table) {
        $table->id();
        $table->foreignId('modulo_id')->constrained()->cascadeOnDelete();
        $table->string('titulo', 200);
        $table->text('descripcion')->nullable();
        $table->unsignedTinyInteger('orden')->default(0);
        $table->boolean('activo')->default(true);
        $table->timestamps();
    });
}
BASH
# Ejecutar todas las migraciones
php artisan migrate

# Reiniciar desde cero (borra y recrea)
php artisan migrate:fresh

# Con datos de prueba
php artisan migrate:fresh --seed

# Ver estado de migraciones
php artisan migrate:status

04
Modelos Eloquent con relaciones
make:model · $fillable · hasMany · belongsTo · $casts
▼

Eloquent es el ORM de Laravel. Cada modelo representa una tabla y cada instancia del modelo es una fila. Las relaciones conectan modelos entre sí.

BASH
# Crear modelos
php artisan make:model Modulo
php artisan make:model Leccion
php artisan make:model Paso
php artisan make:model Ejercicio
php artisan make:model Progreso
PHP
// app/Models/Modulo.php
class Modulo extends Model
{
    protected $fillable = [
        'nombre', 'descripcion', 'icono', 'orden', 'activo',
    ];

    protected function casts(): array
    {
        return [
            'activo' => 'boolean',
            'orden'  => 'integer',
        ];
    }

    // Relación: un módulo tiene muchas lecciones
    public function lecciones(): HasMany
    {
        return $this->hasMany(Leccion::class)->orderBy('orden');
    }
}

// app/Models/Leccion.php
class Leccion extends Model
{
    protected $fillable = [
        'modulo_id', 'titulo', 'descripcion', 'orden', 'activo',
    ];

    // Una lección pertenece a un módulo
    public function modulo(): BelongsTo
    {
        return $this->belongsTo(Modulo::class);
    }

    // Una lección tiene muchos pasos y ejercicios
    public function pasos(): HasMany
    {
        return $this->hasMany(Paso::class)->orderBy('orden');
    }

    public function ejercicios(): HasMany
    {
        return $this->hasMany(Ejercicio::class)->orderBy('orden');
    }
}

05
Rutas — web.php
Route::get · Route::resource · middleware auth · grouping
▼

Las rutas en Laravel mapean URLs a controladores. El middleware auth protege rutas: si no estás autenticado, Laravel te redirige al login.

PHP
// routes/web.php
use App\Http\Controllers\ModuloController;
use App\Http\Controllers\LeccionController;
use App\Http\Controllers\ProgresoController;

// Ruta pública: landing page
Route::get('/', function () {
    return view('welcome');
});

// Rutas protegidas: solo usuarios autenticados
Route::middleware('auth')->group(function () {

    // Dashboard del estudiante
    Route::get('/dashboard', function () {
        return view('dashboard');
    })->name('dashboard');

    // Módulos: index y show
    Route::resource('modulos', ModuloController::class)
        ->only(['index', 'show']);

    // Lecciones: solo show
    Route::resource('lecciones', LeccionController::class)
        ->only(['show']);

    // Registrar progreso
    Route::post('/progreso/{leccion}', [ProgresoController::class, 'marcar'])
        ->name('progreso.marcar');
});

// Rutas de autenticación (Breeze las genera)
require __DIR__.'/auth.php';

06
Controladores
make:controller · index() · show() · Eloquent queries · with()
▼

Los controladores reciben la petición HTTP, consultan los modelos y pasan los datos a la vista. Son el puente entre la URL y el HTML.

BASH
php artisan make:controller ModuloController
php artisan make:controller LeccionController
php artisan make:controller ProgresoController
PHP
// app/Http/Controllers/ModuloController.php
class ModuloController extends Controller
{
    // GET /modulos — lista de módulos activos
    public function index(): View
    {
        $modulos = Modulo::where('activo', true)
            ->orderBy('orden')
            ->withCount('lecciones')  // agrega lecciones_count
            ->get();

        return view('modulos.index', compact('modulos'));
    }

    // GET /modulos/{modulo} — detalle con lecciones
    public function show(Modulo $modulo): View
    {
        // Eager loading: carga lecciones en una sola query
        $modulo->load(['lecciones' => fn($q) => $q->where('activo', true)]);

        return view('modulos.show', compact('modulo'));
    }
}

// app/Http/Controllers/ProgresoController.php
class ProgresoController extends Controller
{
    // POST /progreso/{leccion}
    public function marcar(Leccion $leccion): RedirectResponse
    {
        Progreso::updateOrCreate(
            ['user_id' => auth()->id(), 'leccion_id' => $leccion->id],
            ['completado_at' => now()]
        );

        return back()->with('success', '¡Lección completada!');
    }
}

Vistas Blade
@extends · @section · @foreach · @if · {{ }} · @csrf
▼

Blade es el motor de plantillas de Laravel. Las directivas como @foreach y @if hacen el PHP más legible. {{ }} escapa el HTML automáticamente (previene XSS).

BLADE
{{-- resources/views/modulos/index.blade.php --}}
@extends('layouts.app')

@section('header')
    <h2 class="font-bold text-xl text-gray-200">Mis Módulos</h2>
@endsection

@section('content')
<div class="max-w-5xl mx-auto py-12 px-4">

    @if (session('success'))
        <div class="mb-4 p-4 bg-green-900/50 text-green-300 rounded-xl">
            {{ session('success') }}
        </div>
    @endif

    <div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6">
        @foreach ($modulos as $modulo)
        <a href="{{ route('modulos.show', $modulo) }}"
           class="block bg-gray-900 border border-gray-800 rounded-2xl p-6
             hover:border-orange-500/40 transition">
            <span class="text-4xl">{{ $modulo->icono }}</span>
            <h3 class="mt-4 font-bold text-lg">{{ $modulo->nombre }}</h3>
            <p class="text-gray-400 text-sm mt-2">{{ $modulo->descripcion }}</p>
            <span class="text-xs text-gray-500 mt-3 block">
                {{ $modulo->lecciones_count }} lecciones
            </span>
        </a>
        @endforeach
    </div>
</div>
@endsection

Laravel Breeze — Autenticación
breeze:install · login · register · auth() · @auth · @guest
▼

Breeze genera todo el sistema de autenticación: login, registro, recuperar contraseña, verificación de email. Ya está instalado en el proyecto, veamos cómo funciona.

BASH
# Instalar Breeze (en un proyecto nuevo)
composer require laravel/breeze --dev
php artisan breeze:install blade
npm install && npm run dev
php artisan migrate
BLADE
{{-- En cualquier vista Blade --}}

{{-- Usuario autenticado --}}
@auth
    <p>Hola, {{ auth()->user()->name }}!</p>
    <a href="{{ route('modulos.index') }}">Ver módulos</a>
@endauth

{{-- Usuario no autenticado --}}
@guest
    <a href="{{ route('login') }}">Ingresar</a>
    <a href="{{ route('register') }}">Registrarse</a>
@endguest

{{-- En PHP (controller): --}}
// Obtener usuario autenticado
$usuario = auth()->user();
$userId  = auth()->id();

// Verificar si está autenticado
if (auth()->check()) { /* ... */ }

Seeders — datos de prueba reales
make:seeder · db:seed · DatabaseSeeder · Eloquent create()
▼

Los seeders llenan la BD con datos iniciales o de prueba. En EduCode usamos seeders para cargar los 5 módulos y sus lecciones sin tener que escribir SQL manualmente.

BASH
php artisan make:seeder ModuloSeeder
php artisan make:seeder LeccionSeeder
PHP
// database/seeders/ModuloSeeder.php
class ModuloSeeder extends Seeder
{
    public function run(): void
    {
        $modulos = [
            ['nombre' => 'Bases de Datos', 'icono' => '🗄️', 'orden' => 1],
            ['nombre' => 'PHP Puro',       'icono' => '🐘', 'orden' => 2],
            ['nombre' => 'HTML & CSS',     'icono' => '🎨', 'orden' => 3],
            ['nombre' => 'Laravel',        'icono' => '⚡', 'orden' => 4],
            ['nombre' => 'Laravel Moonshine', 'icono' => '🌙', 'orden' => 5],
        ];

        foreach ($modulos as $data) {
            Modulo::create($data + ['activo' => true]);
        }
    }
}

// database/seeders/DatabaseSeeder.php
public function run(): void
{
    // Usuario de prueba
    User::factory()->create([
        'name'  => 'Estudiante Demo',
        'email' => 'demo@cefit.edu.co',
    ]);

    // Datos de la plataforma
    $this->call([
        ModuloSeeder::class,
        LeccionSeeder::class,
    ]);
}

Deploy en Railway
railway up · variables de entorno · migrate --force · producción
▼

Railway es la plataforma de deploy que usamos. El proyecto en Railway ya tiene su propia BD MySQL con credenciales diferentes al entorno local.

⚠️
Railway tiene plan de pago activo. El deploy puede generar costos. No hagas deploys de prueba innecesarios. Coordina con el instructor antes de hacer railway up.
BASH
# Instalar Railway CLI
npm install -g @railway/cli

# Autenticarse
railway login

# Deployar el proyecto
railway up

# Ejecutar migraciones en producción
railway run php artisan migrate --force

# Ejecutar seeders en producción
railway run php artisan db:seed --force

# Ver logs del deploy
railway logs

Variables de entorno en Railway

Configura estas en el dashboard de Railway (Settings → Variables):

  • APP_NAME = "EduCode CEFIT"
  • APP_ENV = production
  • APP_DEBUG = false
  • DB_* = credenciales MySQL de Railway
  • SESSION_DRIVER = database
