01
Instalación de Moonshine
composer · artisan moonshine:install · admin user · /admin
▼

Moonshine v3 es un paquete que genera un panel de administración completo sobre Laravel, con su propio sistema de auth, UI y componentes. Es independiente de Breeze.

Terminal
# 1. Instalar el paquete
composer require moonshine/moonshine

# 2. Publicar y configurar Moonshine
php artisan moonshine:install

# El comando pregunta:
# - ¿Instalar migraciones de Moonshine? → yes
# - ¿Crear usuario administrador? → yes
# - Nombre, Email, Password del admin

# 3. Ejecutar migraciones
php artisan migrate

# 4. Acceder al panel
# http://tu-dominio/admin
🌙
Moonshine crea su propia tabla moonshine_users separada de la tabla users de Laravel. El admin del panel y los usuarios de la app son independientes por defecto.

02
Resources — el corazón de Moonshine
make:resource · Model · fields() · indexFields · formFields
▼

Un Resource en Moonshine equivale a un CRUD completo para un modelo. Con un solo archivo obtienes listado, creación, edición y eliminación.

Terminal
# Crear resource para el modelo Producto
php artisan moonshine:resource ProductoResource

# Resource con Model asociado automáticamente
php artisan moonshine:resource ProductoResource --model=Producto
PHP — ProductoResource.php
<?php
namespace App\MoonShine\Resources;

use App\Models\Producto;
use MoonShine\Resources\ModelResource;
use MoonShine\Decorations\Block;
use MoonShine\Fields\ID;
use MoonShine\Fields\Text;
use MoonShine\Fields\Number;
use MoonShine\Fields\Switcher;

class ProductoResource extends ModelResource
{
    // Modelo asociado
    protected string $model = Producto::class;

    // Título del resource en el panel
    protected string $title = 'Productos';

    // Columna que se muestra en relaciones
    protected string $column = 'nombre';

    // Campos en el LISTADO y en el FORMULARIO
    public function fields(): array
    {
        return [
            Block::make('Información del Producto', [
                ID::make()->sortable(),

                Text::make('Nombre', 'nombre')
                    ->required()
                    ->sortable()
                    ->searchable(),

                Number::make('Precio', 'precio')
                    ->required()
                    ->min(0)
                    ->step(100),

                Number::make('Stock', 'stock')
                    ->min(0)
                    ->default(0),

                Switcher::make('Activo', 'activo')
                    ->default(true),
            ]),
        ];
    }
}

Registrar el Resource en el menú

PHP — MoonShineServiceProvider.php
use App\MoonShine\Resources\ProductoResource;
use App\MoonShine\Resources\ClienteResource;
use App\MoonShine\Resources\PedidoResource;
use MoonShine\Menu\MenuGroup;
use MoonShine\Menu\MenuItem;

protected function menu(): array
{
    return [
        MenuItem::make('Dashboard', new DashboardPage(), 'heroicons.home'),

        MenuGroup::make('Catálogo', [
            MenuItem::make('Productos', new ProductoResource(), 'heroicons.cube'),
        ]),

        MenuGroup::make('Ventas', [
            MenuItem::make('Clientes', new ClienteResource(), 'heroicons.users'),
            MenuItem::make('Pedidos', new PedidoResource(), 'heroicons.shopping-cart'),
        ]),
    ];
}

Los Fields son los bloques de construcción de un Resource. Moonshine incluye más de 30 tipos de campo listos para usar.

PHP — Uso de múltiples Fields
use MoonShine\Fields\Text;
use MoonShine\Fields\Select;
use MoonShine\Fields\Enum;
use MoonShine\Fields\Date;
use MoonShine\Fields\Image;
use MoonShine\Fields\Textarea;

public function fields(): array
{
    return [
        ID::make()->sortable(),

        Text::make('Nombre', 'nombre')
            ->required()
            ->sortable()
            ->searchable()
            ->placeholder('Ej: Mochila escolar'),

        Textarea::make('Descripción', 'descripcion')
            ->hideOnIndex(),    // solo en formulario

        Number::make('Precio (COP)', 'precio')
            ->min(0)->step(500)->sortable(),

        // Enum column de la BD
        Select::make('Estado', 'estado')
            ->options([
                'pendiente'  => 'Pendiente',
                'enviado'    => 'Enviado',
                'entregado'  => 'Entregado',
                'cancelado'  => 'Cancelado',
            ])
            ->badge(fn($val) => match($val) {
                'pendiente'  => 'warning',
                'enviado'    => 'info',
                'entregado'  => 'success',
                'cancelado'  => 'danger',
                default      => 'default',
            }),

        Date::make('Fecha', 'created_at')
            ->format('d/m/Y')
            ->withTime()
            ->showOnForm(false),

        Image::make('Foto', 'imagen')
            ->disk('public')
            ->dir('productos')
            ->nullable(),

        Switcher::make('Activo', 'activo'),
    ];
}
04
Filtros y búsqueda
filters() · search · sortable · scopes en Resource
▼
PHP — Filtros en el Resource
use MoonShine\Fields\Text;
use MoonShine\Fields\Select;
use MoonShine\Fields\RangeSlider;
use MoonShine\Fields\DateRange;

// Columnas buscables (barra de búsqueda)
protected array $search = ['nombre', 'descripcion'];

// Filtros del panel lateral
public function filters(): array
{
    return [
        Text::make('Nombre', 'nombre')
            ->like('nombre'),       // LIKE %valor%

        Select::make('Estado', 'estado')
            ->options([
                'pendiente' => 'Pendiente',
                'enviado'   => 'Enviado',
                'entregado' => 'Entregado',
            ])
            ->nullable(),

        RangeSlider::make('Precio', 'precio')
            ->fromTo('precio_min', 'precio_max')
            ->min(0)->max(500000),

        DateRange::make('Fecha de registro', 'created_at')
            ->fromTo('fecha_desde', 'fecha_hasta'),

        Switcher::make('Solo activos', 'activo')
            ->default(true),
    ];
}

// Query por defecto (ordenar por nombre)
public function query(): Builder
{
    return parent::query()->orderBy('nombre');
}
05
Relaciones en Moonshine
BelongsTo · HasMany · BelongsToMany · MorphMany
▼
PHP — Relaciones en Resource
use MoonShine\Fields\Relationships\BelongsTo;
use MoonShine\Fields\Relationships\HasMany;
use MoonShine\Fields\Relationships\BelongsToMany;

// En PedidoResource — relación ManyToOne
BelongsTo::make('Cliente', 'cliente', fn() => new ClienteResource())
    ->required()
    ->searchable(),   // búsqueda por nombre en dropdown

BelongsTo::make('Producto', 'producto', fn() => new ProductoResource())
    ->nullable()
    ->placeholder('Seleccionar producto'),


// En ClienteResource — ver pedidos del cliente
HasMany::make('Pedidos', 'pedidos', fn() => new PedidoResource())
    ->hideOnIndex(),  // solo en la vista detalle del cliente


// Muchos a muchos — Estudiante ↔ Cursos
BelongsToMany::make('Cursos', 'cursos', fn() => new CursoResource())
    ->withPivot([
        Number::make('Nota', 'nota'),
        Select::make('Estado', 'estado')
            ->options(['activo' => 'Activo', 'retirado' => 'Retirado']),
    ])
✅
El método searchable() en un BelongsTo activa la búsqueda por AJAX en el dropdown — ideal cuando hay cientos de registros y no quieres cargarlos todos.
06
Actions — acciones personalizadas
ItemAction · BulkAction · make:action · confirmación
▼

Las Actions permiten ejecutar lógica personalizada desde el panel: exportar, cambiar estado, enviar correos, etc.

Terminal
php artisan moonshine:action MarcarEnviadoAction
PHP — Action individual (por fila)
use MoonShine\Actions\ItemAction;
use MoonShine\ActionButtons\ActionButton;

// En el Resource, definir actions()
public function itemActions(): array
{
    return [
        ItemAction::make(
            'Marcar Enviado',
            function(Pedido $pedido) {
                $pedido->update(['estado' => 'enviado']);
            }
        )
        ->icon('heroicons.truck')
        ->withConfirm('¿Confirmar envío?')
        ->canSee(fn(Pedido $p) => $p->estado === 'pendiente'),
    ];
}

// Bulk Action — aplica a múltiples seleccionados
public function bulkActions(): array
{
    return [
        BulkAction::make(
            'Marcar todos Entregados',
            function(Collection $items) {
                $items->each->update(['estado' => 'entregado']);
            }
        )
        ->icon('heroicons.check-circle')
        ->withConfirm(),
    ];
}
07
Dashboard y páginas personalizadas
make:page · Metrics · Cards · Charts · Dashboard
▼
Terminal
php artisan moonshine:page Dashboard
PHP — Dashboard.php (página personalizada)
use MoonShine\Components\FlexibleRender;
use MoonShine\Decorations\Grid;
use MoonShine\Decorations\Column;
use MoonShine\Metrics\ValueMetric;
use MoonShine\Metrics\DonutChartMetric;

class Dashboard extends Page
{
    public function components(): array
    {
        return [
            Grid::make([

                Column::make([
                    ValueMetric::make('Total Clientes')
                        ->value(Cliente::count())
                        ->icon('heroicons.users')
                        ->columnSpan(4),

                    ValueMetric::make('Pedidos Hoy')
                        ->value(
                            Pedido::whereDate('created_at', today())->count()
                        )
                        ->icon('heroicons.shopping-bag')
                        ->columnSpan(4),

                    ValueMetric::make('Ingresos del mes')
                        ->value(
                            '$' . number_format(
                                Pedido::thisMonth()->sum('total'), 0, ',', '.'
                            )
                        )
                        ->columnSpan(4),
                ]),

                Column::make([
                    DonutChartMetric::make('Pedidos por Estado')
                        ->values([
                            'Pendientes'  => Pedido::where('estado','pendiente')->count(),
                            'Enviados'    => Pedido::where('estado','enviado')->count(),
                            'Entregados'  => Pedido::where('estado','entregado')->count(),
                        ])
                        ->columnSpan(6),
                ]),
            ]),
        ];
    }
}

Deploy con Moonshine en Railway
assets · storage · admin user en producción
▼
Terminal — Checklist de deploy
# 1. Publicar assets de Moonshine (CSS/JS del panel)
php artisan moonshine:publish

# 2. Subir a Railway
railway up

# 3. Migrar (incluye tablas de Moonshine)
railway run php artisan migrate --force

# 4. Crear usuario admin en producción
railway run php artisan moonshine:user

# 5. Vincular storage (imágenes subidas)
railway run php artisan storage:link

# 6. Cachear para producción
railway run php artisan config:cache
railway run php artisan route:cache
railway run php artisan view:cache
railway run php artisan moonshine:cache
⚠️
Railway no persiste el filesystem entre deploys. Las imágenes subidas desde Moonshine se perderán. Para producción real usa S3 o Cloudinary como disco de storage.
✅
El panel de Moonshine queda accesible en https://tu-dominio.up.railway.app/admin. Puedes cambiar el prefijo en config/moonshine.php con la clave 'prefix'.
