01
Instalación y primer proyecto
composer · laravel new · Herd · artisan serve
▼

Laravel 11 requiere PHP 8.2+ y Composer. Con Laravel Herd en Windows el proceso es automático.

Terminal
# Verificar versiones
php --version       # PHP 8.2+
composer --version

# Crear nuevo proyecto Laravel 11
composer create-project laravel/laravel mi-proyecto
# O con el instalador de Laravel
laravel new mi-proyecto

# Entrar al proyecto
cd mi-proyecto

# Iniciar servidor de desarrollo
php artisan serve
# Acceder en: http://127.0.0.1:8000

# Ver todos los comandos disponibles
php artisan list

# Ver versión de Laravel
php artisan --version
🛠️
Con Laravel Herd en Windows basta con crear el proyecto en la carpeta ~/Herd y ya queda disponible en http://mi-proyecto.test automáticamente, sin artisan serve.

Estructura del proyecto
app/ · routes/ · resources/ · database/ · public/
▼
Carpeta / ArchivoQué contiene
app/Models/Clases Eloquent — representan las tablas de BD
app/Http/Controllers/Lógica de negocio y manejo de requests
app/Http/Middleware/Filtros que interceptan requests (auth, cors…)
routes/web.phpRutas del navegador (HTML, sesiones)
routes/api.phpRutas de API (sin sesiones, responden JSON)
resources/views/Plantillas Blade (.blade.php)
database/migrations/Scripts de creación y modificación de tablas
database/seeders/Datos de prueba para popular la BD
public/Único directorio accesible desde el navegador
config/Configuración de la app (mail, database, cache…)
.envVariables de entorno locales (NO subir a Git)
artisanCLI de Laravel para generar código y ejecutar tareas
✅
En Laravel 11 el archivo bootstrap/app.php reemplaza a Kernel.php para registrar middlewares y providers. La estructura es más limpia que en versiones anteriores.
03
Rutas (Routes)
web.php · Route::get · parámetros · Route::resource · middleware
▼
PHP — routes/web.php
<?php
use Illuminate\Support\Facades\Route;
use App\Http\Controllers\ProductoController;

// Ruta simple → closure
Route::get('/', function () {
    return view('welcome');
});

// Ruta → Controller@método
Route::get('/productos', [ProductoController::class, 'index']);
Route::post('/productos', [ProductoController::class, 'store']);

// Parámetro en la URL
Route::get('/productos/{id}', [ProductoController::class, 'show']);

// Parámetro opcional
Route::get('/buscar/{termino?}', function($termino = 'todo') {
    return "Buscando: $termino";
});

// Resource routes — genera 7 rutas CRUD automáticamente
Route::resource('productos', ProductoController::class);

// Solo algunas acciones
Route::resource('categorias', CategoriaController::class)
    ->only(['index', 'show']);

// Rutas protegidas por auth
Route::middleware('auth')->group(function () {
    Route::resource('pedidos', PedidoController::class);
    Route::get('/perfil', [PerfilController::class, 'index']);
});

// Ver todas las rutas registradas
// php artisan route:list

04
Controladores
make:controller · Request · Response · Form Request
▼
Terminal
# Crear controlador básico
php artisan make:controller ProductoController

# Controlador resource (con todos los métodos CRUD)
php artisan make:controller ProductoController --resource

# Con Model asociado
php artisan make:controller ProductoController --resource --model=Producto

# Form Request para validación
php artisan make:request StoreProductoRequest
PHP — ProductoController.php
<?php
namespace App\Http\Controllers;

use App\Models\Producto;
use Illuminate\Http\Request;

class ProductoController extends Controller {

    // GET /productos — lista todos
    public function index() {
        $productos = Producto::orderBy('nombre')->paginate(10);
        return view('productos.index', compact('productos'));
    }

    // GET /productos/{id} — muestra uno
    public function show(Producto $producto) {
        // Route Model Binding: Laravel busca el Producto automáticamente
        return view('productos.show', compact('producto'));
    }

    // POST /productos — guarda nuevo
    public function store(Request $request) {
        $validado = $request->validate([
            'nombre' => 'required|string|max:100',
            'precio' => 'required|numeric|min:0',
            'stock'  => 'required|integer|min:0',
        ]);

        Producto::create($validado);
        return redirect()->route('productos.index')
                        ->with('success', 'Producto creado.');
    }

    // DELETE /productos/{id}
    public function destroy(Producto $producto) {
        $producto->delete();
        return back()->with('success', 'Eliminado correctamente.');
    }
}

Vistas con Blade

▼
Blade — layouts/app.blade.php
<!DOCTYPE html>
<html lang="es">

    <meta charset="UTF-8">

</head>

    <main>

            <div class="alerta-ok">{{ session('success') }}</div>
        @endif

        @yield('contenido')
    </main>
</body>
</html>
Blade — productos/index.blade.php
@extends('layouts.app')

@section('titulo', 'Productos')

@section('contenido')
    <h1>Lista de Productos</h1>

    @forelse ($productos as $producto)
        <div>
            <h3>{{ $producto->nombre }}</h3>
            <p>$ {{ number_format($producto->precio, 0, ',', '.') }}</p>

            {{-- Formulario DELETE con method spoofing --}}
            <form action="{{ route('productos.destroy', $producto) }}"
                  method="POST">
                @csrf
                @method('DELETE')
                <button type="submit">Eliminar</button>
            </form>
        </div>
    @empty
        <p>No hay productos registrados.</p>
    @endforelse

    {{-- Paginación --}}
    {{ $productos->links() }}
@endsection
⚠️
{{ }} escapa HTML automáticamente (seguro). {!! !!} muestra HTML sin escapar — úsalo solo cuando confíes en el contenido.

Migraciones y base de datos
make:migration · Schema · migrate · rollback · seeder
▼
Terminal
# Crear migración
php artisan make:migration create_productos_table
php artisan make:migration add_stock_to_productos_table

# Ejecutar migraciones pendientes
php artisan migrate

# Revertir última migración
php artisan migrate:rollback

# Borrar todo y migrar de nuevo (¡borra datos!)
php artisan migrate:fresh

# Con seeders
php artisan migrate:fresh --seed
PHP — create_productos_table.php
public function up(): void
{
    Schema::create('productos', function (Blueprint $table) {
        $table->id();                                     // id BIGINT AUTO_INCREMENT PK
        $table->string('nombre', 100);
        $table->decimal('precio', 10, 2);
        $table->unsignedInteger('stock')->default(0);
        $table->boolean('activo')->default(true);
        $table->timestamps();                             // created_at, updated_at
    });
}

// Migración con FK
Schema::create('pedidos', function (Blueprint $table) {
    $table->id();
    $table->foreignId('cliente_id')
           ->constrained('clientes')
           ->cascadeOnDelete();     // ON DELETE CASCADE
    $table->foreignId('producto_id')
           ->nullable()
           ->constrained('productos')
           ->nullOnDelete();        // ON DELETE SET NULL
    $table->unsignedInteger('cantidad')->default(1);
    $table->enum('estado', ['pendiente','enviado','entregado','cancelado'])
           ->default('pendiente');
    $table->timestamps();
});
07
Eloquent ORM
Model · $fillable · where · create · update · delete · scopes
▼
Terminal
php artisan make:model Producto
php artisan make:model Producto -m   # con migración
php artisan make:model Producto -mcr # + migración + controller resource
PHP — Producto.php (Model)
class Producto extends Model {
    protected $fillable = ['nombre', 'precio', 'stock', 'activo'];

    protected function casts(): array {
        return ['activo' => 'boolean', 'precio' => 'decimal:2'];
    }

    // Relación: un producto tiene muchos pedidos
    public function pedidos() {
        return $this->hasMany(Pedido::class);
    }

    // Scope local: solo productos activos
    public function scopeActivo($query) {
        return $query->where('activo', true);
    }
}
PHP — Consultas Eloquent
// Obtener todos
$productos = Producto::all();

// Con condiciones
$caros = Producto::where('precio', '>', 50000)
                  ->orderBy('nombre')
                  ->paginate(10);

// Usando scope
$activos = Producto::activo()->get();

// Buscar por PK (lanza 404 si no existe)
$p = Producto::findOrFail(5);

// Crear
$nuevo = Producto::create([
    'nombre' => 'Mochila',
    'precio' => 85000,
    'stock'  => 15,
]);

// Actualizar
$p->update(['precio' => 90000]);

// Eliminar (soft delete si usa SoftDeletes)
$p->delete();

// firstOrCreate: busca o crea
Producto::firstOrCreate(
    ['nombre' => 'Lapicero'],
    ['precio' => 1200, 'stock' => 200]
);

Autenticación con Breeze
login · register · reset password · middleware auth
▼
Terminal — Instalar Breeze
# 1. Instalar el paquete
composer require laravel/breeze --dev

# 2. Publicar scaffolding (Blade + Tailwind + Alpine.js)
php artisan breeze:install blade

# 3. Instalar dependencias de Node
npm install
npm run dev

# 4. Migrar (crea tabla users y password_reset_tokens)
php artisan migrate

Breeze genera automáticamente:

  • Rutas: /login, /register, /forgot-password, /reset-password
  • Vistas Blade en resources/views/auth/
  • Controllers en app/Http/Controllers/Auth/
  • Middleware auth listo para proteger rutas

Relaciones Eloquent
hasMany · belongsTo · belongsToMany · with (eager loading)
▼
PHP — Relaciones en modelos
// Cliente hasMany Pedidos
class Cliente extends Model {
    public function pedidos() {
        return $this->hasMany(Pedido::class);
    }
}

// Pedido belongsTo Cliente
class Pedido extends Model {
    public function cliente() {
        return $this->belongsTo(Cliente::class);
    }
    public function producto() {
        return $this->belongsTo(Producto::class);
    }
}

// Relación muchos a muchos (tabla pivot)
class Estudiante extends Model {
    public function cursos() {
        return $this->belongsToMany(Curso::class)
                      ->withPivot('nota', 'estado')
                      ->withTimestamps();
    }
}
PHP — Uso de relaciones
// Eager loading — evita N+1 queries
$pedidos = Pedido::with(['cliente', 'producto'])->get();

// Usar la relación en Blade
@foreach ($pedidos as $pedido)
    {{ $pedido->cliente->nombre }}
    {{ $pedido->producto->nombre ?? 'Sin producto' }}
@endforeach

// Contar relaciones
$clientes = Cliente::withCount('pedidos')->get();
// $cliente->pedidos_count

// Crear registro hijo desde el padre
$cliente->pedidos()->create([
    'producto_id' => 2,
    'cantidad'    => 3,
]);

Deploy en Railway
railway CLI · variables de entorno · migrate · dominio
▼
Terminal — Deploy paso a paso
# 1. Instalar Railway CLI
npm install -g @railway/cli

# 2. Iniciar sesión
railway login

# 3. Vincular proyecto existente o crear nuevo
railway link
# o
railway init

# 4. Subir el código
railway up

# 5. Ejecutar migraciones en producción
railway run php artisan migrate --force

# 6. Seeders si es necesario
railway run php artisan db:seed --force

# 7. Optimizar para producción
railway run php artisan config:cache
railway run php artisan route:cache
railway run php artisan view:cache
