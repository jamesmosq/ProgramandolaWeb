01
Instalación de Laravel 13
PHP 8.3+ · composer · laravel new · Herd en Windows
▼

Laravel 13 requiere PHP 8.3 o superior. Con Laravel Herd en Windows el proceso es automático y no necesitas configurar Apache ni Nginx.

🆕
Laravel 13 (lanzado 17 marzo 2026) — Mínimo PHP 8.3, incluye Laravel AI SDK estable, JSON:API resources, soporte nativo de passkeys y atributos PHP en lugar de propiedades de clase.

Verificar requisitos

Terminal
# Verificar versiones instaladas
php --version      # Debe ser 8.3 o superior
composer --version
node --version
npm --version

Crear proyecto Laravel 13

Terminal
# Opción A: con el instalador de Laravel (recomendado)
laravel new mi-proyecto

# Opción B: con Composer explícito en Laravel 13
composer create-project laravel/laravel:^13.0 mi-proyecto

# Entrar al directorio
cd mi-proyecto

# Levantar servidor de desarrollo
php artisan serve
# Disponible en: http://127.0.0.1:8000
✅
Con Laravel Herd en Windows: crea el proyecto dentro de C:\Users\TuUsuario\Herd\ y queda disponible en http://mi-proyecto.test automáticamente. No necesitas php artisan serve.

Confirmar versión instalada

Terminal
php artisan --version

Configuración del proyecto
.env · APP_KEY · DB · bootstrap/app.php
▼

Toda la configuración de entorno vive en el archivo .env. Laravel 13 usa bootstrap/app.php para registrar middleware y providers, reemplazando el antiguo Kernel.php.

Archivo .env — configuración mínima

APP_NAME="Programando la Web"
APP_ENV=local
APP_KEY=                    # se genera automáticamente
APP_DEBUG=true
APP_URL=http://programando.test

APP_LOCALE=es
APP_FALLBACK_LOCALE=es
APP_FAKER_LOCALE=es_Col

DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=programando
DB_USERNAME=postgres
DB_PASSWORD=base1234

SESSION_DRIVER=file
CACHE_STORE=database
QUEUE_CONNECTION=database

Generar APP_KEY si no existe

Terminal
php artisan key:generate

bootstrap/app.php — nuevo en Laravel 11+

PHP — bootstrap/app.php
use Illuminate\Foundation\Application;
use Illuminate\Foundation\Configuration\Exceptions;
use Illuminate\Foundation\Configuration\Middleware;

return Application::configure(basePath: dirname(__DIR__))
    ->withRouting(
        web: __DIR__.'/../routes/web.php',
        commands: __DIR__.'/../routes/console.php',
        health: '/up',
    )
    ->withMiddleware(function (Middleware $middleware) {
        // Registrar middleware personalizado aquí
    })
    ->withExceptions(function (Exceptions $exceptions) {
        // Manejo de excepciones personalizado
    })->create();
⚠️
El archivo .env nunca va en Git. Está en .gitignore por defecto. Las variables de producción van directamente en Railway o en el servidor.

05
Definir las rutas
Route::resource · Route::get · middleware · route:list
▼

Todas las rutas web van en routes/web.php. Route::resource genera las 7 rutas CRUD en una sola línea.

PHP — routes/web.php
<?php
use Illuminate\Support\Facades\Route;
use App\Http\Controllers\CursoController;

// Landing page pública
Route::get('/', function () {
    return view('welcome');
})->name('home');

// CRUD de cursos — genera 7 rutas automáticamente
// Opción A: público (sin login)
Route::resource('cursos', CursoController::class);

// Opción B: protegido con auth
Route::middleware('auth')->group(function() {
    Route::resource('cursos', CursoController::class);
});

// Verificar rutas generadas
// php artisan route:list

Modelo Eloquent
make:model · $fillable · casts · atributos PHP 13
▼
Terminal
# Crear modelo solo
php artisan make:model Curso

# Crear modelo + migración + controller resource + seeder + factory
php artisan make:model Curso -mcsf --resource
PHP — app/Models/Curso.php
<?php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Curso extends Model
{
    // Columnas que se pueden asignar masivamente
    protected $fillable = [
        'nombre',
        'descripcion',
        'instructor',
        'horas',
        'nivel',
        'activo',
    ];

    // Conversión automática de tipos
    protected function casts(): array
    {
        return [
            'activo' => 'boolean',
            'horas'  => 'integer',
        ];
    }

    // Scope: solo cursos activos
    public function scopeActivo($query)
    {
        return $query->where('activo', true);
    }
}

/* ALTERNATIVA: atributos PHP de Laravel 13
#[Table('cursos')]
#[Fillable('nombre', 'descripcion', 'instructor', 'horas', 'nivel', 'activo')]
class Curso extends Model {}
*/
🆕
Laravel 13: puedes usar atributos PHP #[Fillable], #[Hidden], #[Table] como alternativa moderna a las propiedades de clase. Ambas formas funcionan.

Controlador Resource completo
index · create · store · show · edit · update · destroy
▼
Terminal
php artisan make:controller CursoController --resource --model=Curso
PHP — app/Http/Controllers/CursoController.php
<?php
namespace App\Http\Controllers;

use App\Models\Curso;
use Illuminate\Http\Request;

class CursoController extends Controller
{
    // GET /cursos — lista todos los cursos
    public function index()
    {
        $cursos = Curso::orderBy('nombre')->paginate(10);
        return view('cursos.index', compact('cursos'));
    }

    // GET /cursos/create — formulario de creación
    public function create()
    {
        return view('cursos.create');
    }

    // POST /cursos — guardar nuevo curso
    public function store(Request $request)
    {
        $datos = $request->validate([
            'nombre'     => 'required|string|max:100',
            'descripcion'=> 'nullable|string',
            'instructor' => 'required|string|max:100',
            'horas'      => 'required|integer|min:1',
            'nivel'      => 'required|in:basico,intermedio,avanzado',
        ]);

        Curso::create($datos);

        return redirect()
            ->route('cursos.index')
            ->with('success', 'Curso creado correctamente.');
    }

    // GET /cursos/{curso} — ver detalle
    public function show(Curso $curso)  // Route Model Binding
    {
        return view('cursos.show', compact('curso'));
    }

    // GET /cursos/{curso}/edit — formulario de edición
    public function edit(Curso $curso)
    {
        return view('cursos.edit', compact('curso'));
    }

    // PUT /cursos/{curso} — actualizar
    public function update(Request $request, Curso $curso)
    {
        $datos = $request->validate([
            'nombre'     => 'required|string|max:100',
            'descripcion'=> 'nullable|string',
            'instructor' => 'required|string|max:100',
            'horas'      => 'required|integer|min:1',
            'nivel'      => 'required|in:basico,intermedio,avanzado',
            'activo'     => 'boolean',
        ]);

        $curso->update($datos);

        return redirect()
            ->route('cursos.index')
            ->with('success', 'Curso actualizado.');
    }

    // DELETE /cursos/{curso} — eliminar
    public function destroy(Curso $curso)
    {
        $curso->delete();

        return back()->with('success', 'Curso eliminado.');
    }
}

Validación con Form Request
make:request · rules() · messages() · @error
▼

En lugar de validar en el controlador, Laravel permite crear clases Form Request dedicadas. Mantiene el controlador limpio y centraliza las reglas.

Terminal
php artisan make:request StoreCursoRequest
php artisan make:request UpdateCursoRequest
PHP — app/Http/Requests/StoreCursoRequest.php
<?php
namespace App\Http\Requests;
use Illuminate\Foundation\Http\FormRequest;

class StoreCursoRequest extends FormRequest
{
    public function authorize(): bool
    {
        return true; // false = solo usuarios autenticados pueden pasar
    }

    public function rules(): array
    {
        return [
            'nombre'      => ['required', 'string', 'max:100'],
            'descripcion' => ['nullable', 'string'],
            'instructor'  => ['required', 'string', 'max:100'],
            'horas'       => ['required', 'integer', 'min:1', 'max:500'],
            'nivel'       => ['required', 'in:basico,intermedio,avanzado'],
        ];
    }

    // Mensajes de error en español
    public function messages(): array
    {
        return [
            'nombre.required'     => 'El nombre del curso es obligatorio.',
            'instructor.required' => 'El instructor es obligatorio.',
            'horas.min'           => 'El curso debe tener al menos 1 hora.',
            'nivel.in'            => 'El nivel debe ser básico, intermedio o avanzado.',
        ];
    }
}

requiredCampo obligatorio, no puede estar vacío
nullablePuede ser null o no enviarse
stringDebe ser texto
integerDebe ser número entero
