01
Crear la base de datos
CREATE DATABASE · USE · CHARACTER SET · COLLATE
▼

Antes de crear cualquier tabla necesitamos un contenedor: la base de datos. En este proyecto usamos el nombre programando, que coincide con el archivo .env de Laravel.

💡
Abre WAMP → phpMyAdmin o el cliente MySQL de tu preferencia. Ejecuta los comandos en el orden indicado.

Comandos

SQL
-- Ver las bases de datos existentes
SHOW DATABASES;

-- Crear la base de datos de la plataforma
CREATE DATABASE programando
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- Seleccionarla para trabajar
USE programando;

-- Verificar que estamos en la BD correcta
SELECT DATABASE();
⚠️
Siempre usa utf8mb4, no utf8. utf8mb4 soporta tildes, ñ y emojis correctamente en MySQL moderno.

02
Tabla modulos
CREATE TABLE · INT · VARCHAR · TINYINT · TIMESTAMP · ENGINE=InnoDB
▼

Los módulos son los bloques principales del curso (BD, PHP, HTML/CSS, Laravel, Moonshine). Esta tabla no tiene claves foráneas — es tabla padre.

🚀
Regla de oro: crear siempre las tablas padre primero, luego las hijas. Si hay FK y la tabla padre no existe, el script fallará.
SQL
CREATE TABLE modulos (
  id          INT UNSIGNED NOT NULL AUTO_INCREMENT,
  nombre      VARCHAR(120) NOT NULL,
  descripcion TEXT NULL,
  icono       VARCHAR(60) NULL,          -- emoji o nombre de ícono
  orden       TINYINT UNSIGNED NOT NULL DEFAULT 0,
  activo      TINYINT(1) NOT NULL DEFAULT 1,
  created_at  TIMESTAMP NULL DEFAULT NULL,
  updated_at  TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

Columnas importantes

  • id — INT UNSIGNED AUTO_INCREMENT: clave primaria auto-numérica positiva
  • activo — TINYINT(1): actúa como booleano (0 = inactivo, 1 = activo)
  • orden — define el orden de aparición en la plataforma
  • created_at / updated_at — timestamps que Laravel maneja automáticamente

03
Tabla lecciones
FOREIGN KEY · REFERENCES · ON DELETE CASCADE
▼

Cada módulo contiene varias lecciones. La columna modulo_id es una clave foránea (FK) que apunta a modulos.id. Si se elimina un módulo, sus lecciones se borran automáticamente (CASCADE).

SQL
CREATE TABLE lecciones (
  id          INT UNSIGNED NOT NULL AUTO_INCREMENT,
  modulo_id   INT UNSIGNED NOT NULL,
  titulo      VARCHAR(200) NOT NULL,
  descripcion TEXT NULL,
  orden       TINYINT UNSIGNED NOT NULL DEFAULT 0,
  activo      TINYINT(1) NOT NULL DEFAULT 1,
  created_at  TIMESTAMP NULL DEFAULT NULL,
  updated_at  TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (id),
  CONSTRAINT fk_lecciones_modulo
    FOREIGN KEY (modulo_id)
    REFERENCES modulos(id)
    ON DELETE CASCADE
    ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
🔗
ON DELETE CASCADE: si borras el módulo "Bases de Datos", todas sus lecciones se eliminan automáticamente. Mantiene la integridad referencial.

04
Tabla pasos
LONGTEXT · ENUM · orden de creación
▼

Cada lección tiene varios pasos de contenido: teoría, bloques de código, tips o advertencias. El campo contenido es LONGTEXT porque puede tener HTML enriquecido.

SQL
CREATE TABLE pasos (
  id         INT UNSIGNED NOT NULL AUTO_INCREMENT,
  leccion_id INT UNSIGNED NOT NULL,
  titulo     VARCHAR(200) NOT NULL,
  contenido  LONGTEXT NOT NULL,
  tipo       ENUM('teoria','codigo','tip','advertencia') NOT NULL DEFAULT 'teoria',
  orden      TINYINT UNSIGNED NOT NULL DEFAULT 0,
  created_at TIMESTAMP NULL DEFAULT NULL,
  updated_at TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (id),
  CONSTRAINT fk_pasos_leccion
    FOREIGN KEY (leccion_id) REFERENCES lecciones(id)
    ON DELETE CASCADE ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

¿Qué es ENUM?

ENUM restringe el campo a una lista de valores válidos. Si intentas insertar un valor que no esté en la lista, MySQL lanza un error. Es ideal para columnas como tipo donde solo hay opciones fijas.

05
Tabla ejercicios
BOOLEAN · TEXT nullable · dificultad ENUM
▼

Al final de cada lección hay ejercicios propuestos. El instructor puede decidir si mostrar la solución o no con la columna solucion_visible.

SQL
CREATE TABLE ejercicios (
  id               INT UNSIGNED NOT NULL AUTO_INCREMENT,
  leccion_id       INT UNSIGNED NOT NULL,
  enunciado        TEXT NOT NULL,
  dificultad       ENUM('basico','intermedio','avanzado') NOT NULL DEFAULT 'basico',
  solucion_visible TINYINT(1) NOT NULL DEFAULT 0,
  solucion         TEXT NULL,
  orden            TINYINT UNSIGNED NOT NULL DEFAULT 0,
  created_at       TIMESTAMP NULL DEFAULT NULL,
  updated_at       TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (id),
  CONSTRAINT fk_ejercicios_leccion
    FOREIGN KEY (leccion_id) REFERENCES lecciones(id)
    ON DELETE CASCADE ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
💡
La columna solucion es TEXT NULL — puede estar vacía si el instructor no ha escrito la solución aún. solucion_visible controla si el estudiante puede verla.

06
Tabla progresos
FK múltiples · UNIQUE KEY · TIMESTAMP nullable
▼

Esta tabla registra qué lecciones completó cada estudiante. La clave UNIQUE(user_id, leccion_id) garantiza que un estudiante no pueda "completar" la misma lección dos veces.

⚠️
Esta tabla referencia users que en este contexto la crea Laravel. Si estás ejecutando solo el SQL (sin Laravel), debes crear primero la tabla users o eliminar esa FK.
SQL
CREATE TABLE progresos (
  id            INT UNSIGNED NOT NULL AUTO_INCREMENT,
  user_id       BIGINT UNSIGNED NOT NULL,     -- mismo tipo que users.id en Laravel
  leccion_id    INT UNSIGNED NOT NULL,
  completado_at TIMESTAMP NULL DEFAULT NULL, -- NULL = no completado aún
  created_at    TIMESTAMP NULL DEFAULT NULL,
  updated_at    TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uq_progreso (user_id, leccion_id),
  CONSTRAINT fk_progresos_leccion
    FOREIGN KEY (leccion_id) REFERENCES lecciones(id)
    ON DELETE CASCADE ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

¿Por qué UNIQUE(user_id, leccion_id)?

La combinación de ambas columnas debe ser única: un estudiante puede tener un único registro de progreso por lección. Sin esta restricción podría registrar la misma lección como completada múltiples veces.

08
SELECT y JOINs
SELECT · WHERE · JOIN · COUNT · GROUP BY · ORDER BY
▼

Con las tablas pobladas, practiquemos las consultas que la plataforma usará para mostrar contenido al estudiante.

SQL
-- 1. Todos los módulos activos ordenados
SELECT id, nombre, icono, orden
FROM  modulos
WHERE activo = 1
ORDER BY orden ASC;

-- 2. Lecciones con el nombre de su módulo (INNER JOIN)
SELECT
  l.id,
  m.nombre AS modulo,
  l.titulo,
  l.orden
FROM       lecciones l
INNER JOIN modulos   m ON l.modulo_id = m.id
ORDER BY   m.orden, l.orden;

-- 3. Cuántas lecciones tiene cada módulo
SELECT
  m.nombre,
  COUNT(l.id) AS total_lecciones
FROM      modulos   m
LEFT JOIN lecciones l ON l.modulo_id = m.id
GROUP BY  m.id, m.nombre
ORDER BY  m.orden;
🔍
INNER JOIN retorna solo las filas que tienen coincidencia en ambas tablas. LEFT JOIN retorna todos de la tabla izquierda aunque no haya coincidencia (útil para contar, incluye módulos sin lecciones).

09
ALTER TABLE — Modificar estructura
ADD COLUMN · MODIFY · DROP COLUMN · AFTER
▼

Cuando la aplicación evoluciona, necesitamos modificar tablas existentes. ALTER TABLE permite agregar, modificar o eliminar columnas sin borrar los datos.

SQL
-- Agregar columna 'duracion_minutos' a lecciones
ALTER TABLE lecciones
  ADD COLUMN duracion_minutos SMALLINT UNSIGNED NULL DEFAULT NULL
  AFTER descripcion;

-- Agregar columna 'imagen_url' a modulos
ALTER TABLE modulos
  ADD COLUMN imagen_url VARCHAR(500) NULL
  AFTER icono;

-- Modificar el tamaño de 'titulo' en lecciones
ALTER TABLE lecciones
  MODIFY COLUMN titulo VARCHAR(255) NOT NULL;

-- Ver la estructura actualizada
DESCRIBE lecciones;
💡
Usa siempre AFTER columna al agregar columnas. Así controlas el orden visual en las consultas y mantiene coherencia con las migraciones de Laravel.

10
Script completo de la plataforma
DROP IF EXISTS · script reproducible · orden correcto
▼

Este es el script maestro que puedes ejecutar desde cero para recrear toda la BD. Usa DROP TABLE IF EXISTS en orden inverso para evitar errores de FK.

⚠️
Este script borra y recrea todas las tablas. Solo úsalo en desarrollo. En producción usa migraciones de Laravel con php artisan migrate.
SQL
-- ============================================
-- EduCode — Script BD completo
-- Ejecutar en orden. Requiere: USE programando;
-- ============================================

SET FOREIGN_KEY_CHECKS = 0;

-- Eliminar en orden hijo → padre
DROP TABLE IF EXISTS progresos;
DROP TABLE IF EXISTS ejercicios;
DROP TABLE IF EXISTS pasos;
DROP TABLE IF EXISTS lecciones;
DROP TABLE IF EXISTS modulos;

SET FOREIGN_KEY_CHECKS = 1;

-- Crear en orden padre → hijo
CREATE TABLE modulos (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  nombre VARCHAR(120) NOT NULL,
  descripcion TEXT NULL,
  icono VARCHAR(60) NULL,
  orden TINYINT UNSIGNED NOT NULL DEFAULT 0,
  activo TINYINT(1) NOT NULL DEFAULT 1,
  created_at TIMESTAMP NULL DEFAULT NULL,
  updated_at TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE lecciones (
  id INT UNSIGNED NOT NULL AUTO_INCREMENT,
  modulo_id INT UNSIGNED NOT NULL,
  titulo VARCHAR(200) NOT NULL,
  descripcion TEXT NULL,
  orden TINYINT UNSIGNED NOT NULL DEFAULT 0,
  activo TINYINT(1) NOT NULL DEFAULT 1,
  created_at TIMESTAMP NULL DEFAULT NULL,
  updated_at TIMESTAMP NULL DEFAULT NULL,
  PRIMARY KEY (id),
  CONSTRAINT fk_lecciones_modulo
    FOREIGN KEY (modulo_id) REFERENCES modulos(id)
    ON DELETE CASCADE ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
