01
Instalar Laravel Moonshine
composer require · moonshine:install · /admin · MoonShineServiceProvider
▼

Moonshine v3 es un panel de administración construido específicamente para Laravel. Se instala como un paquete de Composer y genera todo su scaffolding automáticamente.

🌙
Moonshine genera su propio sistema de autenticación separado del de la aplicación. El admin tendrá su propio usuario y contraseña, independiente de los estudiantes.
BASH
# 1. Instalar el paquete
composer require moonshine/moonshine

# 2. Ejecutar el instalador
php artisan moonshine:install

# El instalador pregunta:
# → Confirm install? (yes)
# → Choose template: default
# → Use migrations? (yes)
# → Create superadmin? (yes)
# → Name / Email / Password del admin

# 3. Ejecutar migraciones de Moonshine
php artisan migrate

# 4. Abrir el panel
# → http://programando.test/admin

¿Qué genera el instalador?

  • app/MoonShine/ — directorio para Resources, Pages y Layouts
  • app/Providers/MoonShineServiceProvider.php — registro de Resources
  • config/moonshine.php — configuración del panel
  • Tabla moonshine_users — usuarios del admin (separada de users)
  • Ruta /admin — acceso al panel con middleware moonshine.auth

02
ModuloResource
moonshine:resource · fields() · indexFields · formFields · Text · Number · Toggle
▼

Un Resource en Moonshine es la clase que define cómo se administra un modelo: qué campos se muestran, en qué formato, con qué validaciones.

BASH
# Crear el Resource para Modulo
php artisan moonshine:resource ModuloResource
PHP
// app/MoonShine/Resources/ModuloResource.php
use MoonShine\Laravel\Resources\ModelResource;
use MoonShine\UI\Fields\ID;
use MoonShine\UI\Fields\Text;
use MoonShine\UI\Fields\Textarea;
use MoonShine\UI\Fields\Number;
use MoonShine\UI\Fields\Toggle;

class ModuloResource extends ModelResource
{
    // Modelo que administra
    protected string $model = Modulo::class;

    // Título del Resource en el panel
    protected string $title = 'Módulos';

    // Columna por la que se ordena por defecto
    protected string $sortColumn = 'orden';

    // Campos del formulario y tabla
    protected function fields(): FieldsContract
    {
        return IndexFields::make([

            ID::make()->sortable(),

            Text::make('Nombre', 'nombre')
                ->required()
                ->sortable(),

            Text::make('Ícono', 'icono')
                ->placeholder('🗄️'),

            Textarea::make('Descripción', 'descripcion')
                ->nullable()
                ->hideOnIndex(),       // no mostrar en la lista

            Number::make('Orden', 'orden')
                ->default(0)
                ->sortable(),

            Toggle::make('Activo', 'activo')
                ->default(true),
        ]);
    }
}

Registrar el Resource

PHP
// app/Providers/MoonShineServiceProvider.php
protected function resources(): array
{
    return [
        ModuloResource::class,
        // Aquí irán los demás Resources
    ];
}

03
LeccionResource + BelongsTo
BelongsTo field · searchable · relación en formulario
▼

El campo BelongsTo de Moonshine genera un select (o búsqueda) para elegir el módulo al que pertenece la lección. Es el equivalente visual de la FK modulo_id.

PHP
// app/MoonShine/Resources/LeccionResource.php
use MoonShine\UI\Fields\BelongsTo;

class LeccionResource extends ModelResource
{
    protected string $model     = Leccion::class;
    protected string $title     = 'Lecciones';
    protected string $sortColumn = 'orden';

    protected function fields(): FieldsContract
    {
        return IndexFields::make([

            ID::make()->sortable(),

            // Selector del módulo padre
            BelongsTo::make(
                'Módulo',           // label
                'modulo',           // relación en el modelo
                'nombre',           // columna a mostrar del módulo
                ModuloResource::class  // Resource relacionado
            )
            ->required()
            ->searchable(),      // buscador en el select

            Text::make('Título', 'titulo')->required(),

            Textarea::make('Descripción', 'descripcion')->nullable()->hideOnIndex(),

            Number::make('Orden', 'orden')->default(0)->sortable(),

            Toggle::make('Activo', 'activo')->default(true),
        ]);
    }
}
💡
El parámetro 'nombre' en BelongsTo indica qué columna del módulo se muestra en el select. Si el módulo tiene un método __toString(), puedes omitir este parámetro.

04
PasoResource
Select · TinyMce · hideOnIndex · campo ENUM · contenido HTML
▼

Los pasos tienen un campo ENUM (tipo) y el contenido puede ser HTML enriquecido. Moonshine tiene un field TinyMce para editores WYSIWYG.

PHP
// app/MoonShine/Resources/PasoResource.php
use MoonShine\UI\Fields\Select;
use MoonShine\UI\Fields\TinyMce;

class PasoResource extends ModelResource
{
    protected string $model     = Paso::class;
    protected string $title     = 'Pasos de Lección';
    protected string $sortColumn = 'orden';

    protected function fields(): FieldsContract
    {
        return IndexFields::make([

            ID::make(),

            BelongsTo::make('Lección', 'leccion', 'titulo', LeccionResource::class)
                ->required()->searchable(),

            Text::make('Título', 'titulo')->required(),

            // Select para el ENUM
            Select::make('Tipo', 'tipo')
                ->options([
                    'teoria'      => '📖 Teoría',
                    'codigo'      => '💻 Código',
                    'tip'         => '💡 Tip',
                    'advertencia' => '⚠️ Advertencia',
                ])
                ->required()
                ->default('teoria'),

            // Editor WYSIWYG para el contenido
            TinyMce::make('Contenido', 'contenido')
                ->required()
                ->hideOnIndex(),

            Number::make('Orden', 'orden')->default(0),
        ]);
    }
}
✏️
TinyMce es un editor WYSIWYG gratuito. Moonshine lo integra sin configuración adicional. El contenido se guarda como HTML en la columna contenido (LONGTEXT).

05
EjercicioResource
Select dificultad · Checkbox/Toggle · Textarea solución nullable
▼

El resource de ejercicios introduce el campo condicional: la solución solo se puede ver cuando solucion_visible = true. En Moonshine podemos hacer campos dependientes.

PHP
// app/MoonShine/Resources/EjercicioResource.php
class EjercicioResource extends ModelResource
{
    protected string $model = Ejercicio::class;
    protected string $title = 'Ejercicios';

    protected function fields(): FieldsContract
    {
        return IndexFields::make([

            ID::make(),

            BelongsTo::make('Lección', 'leccion', 'titulo', LeccionResource::class)
                ->required()->searchable(),

            Textarea::make('Enunciado', 'enunciado')->required(),

            Select::make('Dificultad', 'dificultad')
                ->options([
                    'basico'      => '🟢 Básico',
                    'intermedio'  => '🟡 Intermedio',
                    'avanzado'    => '🔴 Avanzado',
                ])
                ->required()
                ->default('basico'),

            Toggle::make('Mostrar solución', 'solucion_visible')
                ->default(false),

            Textarea::make('Solución', 'solucion')
                ->nullable()
                ->hideOnIndex()
                ->hint('Solo visible si "Mostrar solución" está activo'),

            Number::make('Orden', 'orden')->default(0),
        ]);
    }
}

06
HasMany — relaciones en el admin
HasMany field · ver hijos desde el padre · relación inline
▼

Con el campo HasMany puedes ver y gestionar las lecciones de un módulo directamente desde la página de detalle del módulo en el admin. Sin tener que ir a otro Resource.

PHP
// Agregar HasMany en ModuloResource
use MoonShine\UI\Fields\HasMany;

protected function fields(): FieldsContract
{
    return IndexFields::make([
        ID::make(),
        Text::make('Nombre', 'nombre')->required(),
        Text::make('Ícono', 'icono'),
        Number::make('Orden', 'orden')->sortable(),
        Toggle::make('Activo', 'activo'),

        // Lecciones del módulo — solo en detalle/edición
        HasMany::make(
            'Lecciones',              // label
            'lecciones',              // relación en el modelo Modulo
            LeccionResource::class   // Resource de lecciones
        )->hideOnIndex(),           // solo visible en la página de edición
    ]);
}
🔗
Al entrar a editar un módulo, verás una tabla inline con sus lecciones. Puedes crear, editar y eliminar lecciones sin salir del módulo. Esto es la potencia de HasMany en Moonshine.

07
Actions — acciones personalizadas
ItemAction · BulkAction · withConfirm · notifySuccess
▼

Las Actions de Moonshine son botones que ejecutan lógica personalizada sobre uno o varios registros. Por ejemplo: activar/desactivar todos los módulos seleccionados.

PHP
// Agregar BulkAction en ModuloResource
use MoonShine\ActionButtons\ActionButton;
use MoonShine\Laravel\ActionButtons\BulkButton;

protected function indexButtons(): ListOf
{
    return parent::indexButtons()->add(

        // Botón masivo: desactivar módulos seleccionados
        BulkButton::make('Desactivar seleccionados')
            ->icon('eye-slash')
            ->withConfirm(
                '¿Desactivar módulos?',
                'Los módulos seleccionados dejarán de verse para los estudiantes.'
            )
            ->method('POST')
            ->url(fn($item) => route('moonshine.modulos.desactivar', $item))
    );
}

// ItemAction en cada fila — duplicar módulo
protected function itemActions(): ListOf
{
    return parent::itemActions()->add(
        ActionButton::make('Activar/Desactivar')
            ->icon('power')
            ->inModal(false)
            ->method('POST')
            ->url(fn($modulo) => route('modulos.toggle', $modulo))
    );
}
⚠️
Las acciones que usan rutas propias necesitan los endpoints definidos en routes/web.php. Para acciones simples puedes usar ActionButton::make()->method('DELETE') y apuntar al controlador de Moonshine.

08
Dashboard con estadísticas
DashboardPage · StatBlock · metrics · resumen de la plataforma
▼

El dashboard es la primera pantalla del admin. Podemos personalizarlo para mostrar estadísticas clave de la plataforma: total de módulos, lecciones, estudiantes registrados y completados.

PHP
// app/MoonShine/Pages/Dashboard.php (generado por Moonshine)
use MoonShine\UI\Components\Metrics\Wrapped\Value;
use App\Models\{Modulo, Leccion, User, Progreso};

class Dashboard extends Page
{
    protected function components(): array
    {
        return [
            // Tarjeta de estadística — total módulos activos
            Value::make(
                'Módulos activos',
                static fn() => Modulo::where('activo', true)->count()
            )
            ->icon('academic-cap')
            ->columnSpan(3),

            // Total lecciones
            Value::make(
                'Total lecciones',
                static fn() => Leccion::where('activo', true)->count()
            )
            ->icon('book-open')
            ->columnSpan(3),

            // Estudiantes registrados
            Value::make(
                'Estudiantes',
                static fn() => User::count()
            )
            ->icon('users')
            ->columnSpan(3),

            // Lecciones completadas hoy
            Value::make(
                'Completadas hoy',
                static fn() => Progreso::whereDate('completado_at', today())->count()
            )
            ->icon('check-circle')
            ->columnSpan(3),
        ];
    }
}
